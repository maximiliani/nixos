# NixOS configurations

## Adding new hosts to SOPS

Make sure that sops is installed. See flake.nix for this and add the sops flake.

Modify the .sops.yaml file in the root of the repository to add the new host to the list of hosts.
Add an arbitrary key for the host (will change later) and modify other key groups as needed.
For example:

```yaml
keys:
  - &<hostname> age1zlmedva0jsdys79d39qgy595clx5zemjzl2w0qpgewq2psj5yggswgltf2
creation_rules:
  - path_regex: ^secrets/<hostname>/.*$
    key_groups:
      - pgp:
          - *admin
        age:
          - *<hostname>
```

Create a new file in secrets/<hostname>/default.yaml and add the secrets for the host.
Important! Make some change when editing with sops edit.
For example:

```bash
mkdir -p secrets/<hostname>
sops edit secrets/<hostname>/default.yaml
```

Replace the arbitrary key for the host in .sops.yaml with the new key generated by sops edit.
See the command line output of sops edit for the new key.

Add sops to configuration.nix:

```nix
{
  sops = {
    defaultSopsFile = self + /secrets/<hostname>/default.yaml;
    age = {
     keyFile = "/var/lib/sops-nix/key.txt";
     generateKey = true;
    };
    secrets = {
      hello = { };
    };
  };
}
```

Nixos-rebuild the host and check that the secrets are available in /run/secrets/hello.
Git add, commit and push the changes to the repository.

On another host git pull the changes and update the keys for the new secrets file:

```bash
export SOPS_AGE_KEY_FILE=/var/lib/sops-nix/key.txt
git pull
find secrets -name "*" -type f -exec sops updatekeys {} -y \;
git commit -am "Update keys for new host" && git push
```
